Variants — how component sets are emitted
What do these do? · Examples

The same Button set (axes Size, State value-only; Badge structural) under each mode:

Off — one def = first variant's tree; combos ride on the refs. What other variants restyle isn't captured.

"components": { "Button": { "node": { … } } }   // first variant only
// ref: { "use": "Button", "variants": { "Size": "Large", "Badge": "on" }, "size": { … } }

Split by structure — structural variants split; value axes still share one def. Use-refs gain a variant pointer.

"components": { "Button": { "variants": {
  "Badge=off": { "node": … },
  "Badge=on":  { "node": … }   // + Badge child
} } }
// ref: { "use": "Button", "variant": "Badge=on", "variants": { … } }

Value table — base node + per-axis deltas (structural variants still split out). Heaviest; best in the panel (not the 3 s Dev Mode budget).

"components": { "Button": {
  "node": { … },                 // default variant = base
  "variantStyles": {
    "Size":  { "Large": { "size": { "width": 64, "height": 64 } } },
    "State": { "Disabled": { "Background: fill": "#E0E0E0" } }
  }
} }
Select one or more frames — output updates automatically. Exports compact nodes (tree + layout/style/colour + bound-variable refs) plus flat colors, textStyles and dimensions catalogs and a components map (Figma components + deduped repeated frames, as { use, props }). Copy the JSON for codegen.
